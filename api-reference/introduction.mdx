The Xenovia API provides programmatic access to runtime governance workflows, including proxy management, runtime-derived identity metadata, policy checks, trace retrieval, and review operations.

## Base URL

```bash
https://api.staging.xenovia.io/api/v1
```

## Design principles

<Tabs>
  <Tab title="REST">
    Resource-oriented endpoints with predictable behavior.
  </Tab>
  <Tab title="JSON">

  </Tab>

## Common use cases

- Creating and managing proxy inventory.
- Reading runtime-derived agent/tool identities for tracing and controls.
- Enforcing and testing policy outcomes programmatically.
- Reviewing trace details for incident response workflows.
- Automating approval or remediation paths in internal systems.

See [Authentication](/api-reference/authentication) and [Errors and Limits](/api-reference/errors-and-limits) before integrating endpoint-level operations.
